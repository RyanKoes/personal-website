Coffee Analysis Repository

Python, NumPy, Pandas, scikit-learn, PyTorch, Matplotlib

I use this repository to analyze electrochemical coffee measurements and related attributes, then build and compare predictive models for targets like HPLC caffeine (Caff), HPLC chlorogenic acids (CGA), and TDS. It’s my end-to-end research workspace for loading and cleaning datasets, generating features (including voltage/time-series “window” selections), training regression and neural network models (including architecture searches), evaluating performance with cross-validation-style splits, and producing plots and summary tables that I can use to interpret results and support my thesis/research writeups.

Image Enhancement Tool

PyTorch, Hugging Face Diffusers & Transformers, Weights & Biases, NumPy, Pillow, SciKit-Learn

This project is an image enhancement tool built around modern diffusion-model ideas: it experiments with using a pretrained latent diffusion model (Stable Diffusion via Hugging Face Diffusers) as a frozen backbone (VAE + UNet + scheduler) and then training a lightweight “ControlNet-style” conditioning module to guide denoising toward higher-quality outputs from a degraded/low-quality input image. The repo includes notebooks for loading and probing pretrained diffusion components and a PyTorch-based training/data pipeline (with on-the-fly degradations, paired LQ/HQ conventions, and optional Weights & Biases logging/sweeps), with the longer-term goal of producing a controllable latent diffusion enhancement pipeline that can be trained on datasets like DIV2K/Kaggle and used for inference to improve image quality.

Coffee Blends Creator

Next.js, React, TypeScript, Tailwind CSS, Vercel

This is a Coffee Blend Builder web app using Next.js (App Router), React, TypeScript, and Tailwind CSS that lets users select a brew method, dial in target flavor intensities with interactive sliders, and add free-form preferences. It implements a serverless API route that integrates with Google Gemini to generate a practical blend recipe (origins/regions, processing, altitude ranges, roast guidance, and ratio breakdown) tailored to the user’s inputs, with input validation, clear error handling, and model fallbacks for reliability. Delivered a clean, accessible UI (ARIA-labeled controls) plus a “Learn” section that explains how roast, processing, altitude, and brew method influence flavor.
